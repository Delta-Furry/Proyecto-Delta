Tema 3: Protocolo HTTP y HTTPS

Introducción a HTTP y HTTPS

HTTP (Hypertext Transfer Protocol) es el protocolo base para la comunicación en la web. Permite la transferencia de documentos HTML, imágenes y otros recursos entre un servidor y un cliente.

Por otro lado, HTTPS es una versión segura de HTTP que utiliza cifrado SSL/TLS para proteger la información transmitida, asegurando la privacidad y la integridad de los datos.

Funcionamiento de HTTP

HTTP opera como un protocolo de solicitud y respuesta. El cliente (normalmente un navegador) envía una solicitud al servidor, y este devuelve una respuesta con el contenido solicitado.

  • Métodos de solicitud: Los métodos más comunes son GET (para obtener datos) y POST (para enviar datos).

  • Códigos de estado: Indican el resultado de la solicitud. Por ejemplo, 200 OK significa éxito, y 404 Not Found indica que el recurso no se encontró.

Beneficios de HTTPS

HTTPS ofrece múltiples ventajas sobre HTTP, incluyendo:

  • Cifrado: Protege la información durante la transferencia.

  • Autenticación: Asegura que estás conectado al servidor correcto.

  • Confianza del usuario: Los navegadores muestran un candado en la barra de direcciones, indicando seguridad.

HTTP/2 y más allá

Con el avance de la tecnología, se han desarrollado nuevas versiones del protocolo, como HTTP/2 y HTTP/3, que ofrecen mejoras significativas en velocidad y seguridad.

HTTP/2 introduce la multiplexación de solicitudes, mientras que HTTP/3 utiliza un nuevo protocolo de transporte llamado QUIC para mejorar la latencia y la fiabilidad.
